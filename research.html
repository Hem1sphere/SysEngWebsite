Review of Existing Solutions

Kaggle is a online platfrom for creating competitions and building kernels for those competitions, and also allows data scientists to learn and communicate via its forum.

AI test platform in our project has similar functionalities except that ours is at a smaller scale, and communications between data scientists is not a must. However, strict prevention of data leak is mandatory according to GDPR. We, therefore, reviewed the design principles of Kaggle and integrated it into our project.

At the beginning stage, we looked into both Python and JavaScript as the main programming language used in NHS PlayGround web app project.

With regard to JavaScript, the Node.js is quite powerful and can provide services from huge number of free tools. As for Python, there is a python-based free and open-source web framework named Django allowing you to quickly and efficiently create a quality Web application. Django is a full stack web development framework can be used to develop full fledged websites in Python. It encourages rapid development and advocates pragmatic and clean code [2]. In this project, we have to using lots of python-based tools and server (e.g. Jupyter). It could be easier for us to use Python in this project without many "pipeline" files.


Choice of Database

As for the database, the options explored were MySQL and SQLite. SQLite is an embedded database and has less features compared to MySQL such as support for concurrency and does not scale to huge amounts of data. However, in our application where we do not foresee a need for concurrency and huge datasets, we decided to go with the simpler option of SQLite which sufficiently serves our needs.


Embeded Coding Environment

- AWS SageMaker VS Azure Machine Learning

For the embedded coding environment, we first looked into using AWS SageMaker and Azure Machine Learning Studio.

The advantages of using AWS SageMaker [3] are:

  • A fully-managed platform that enables developers and data scientists to quickly and easily build, train, and deploy machine learning models at any scale.
  • Removes all the barriers that typically slow down developers who want to use machine learning.
  • Makes it easy to build ML models and get them ready for training by providing everything you need to quickly connect to your training data, and to select and optimize the best algorithm and framework for your application
  • For training, users can begin training their model with a single click in the Amazon SageMaker console.

However, AWS SageMaker cannot be embedded into the web app.

The benefits Azure Machine Learning Studio [4] provide are:

  • Build and train machine-learning models faster, and easily deploy to the cloud or the edge with Azure Machine Learning service.
  • Use the latest open source technologies such as TensorFlow, PyTorch or Jupyter.
  • Experiment locally and then quickly scale up or out with large GPU-enabled clusters in the cloud.
  • Track your experiments, manage models and easily deploy with integrated CI/CD tooling.

However, Azure Machine Learninf Studio is not available in UK region.


- Comparisons among Jupyter Projects

Then, we thought of embedding the Jupyter notebook directly on the web app. The Jupyter Notebook is an open-source web application that allows you to create and share documents that contain live code, equations, visualizations and narrative text. Uses include: data cleaning and transformation, numerical simulation, statistical modeling, data visualization, machine learning, and much more. However, Jupyter notebooks do not support multiple users and can hardly be deployed on a remote server.

Based on our research, JupyterHub is a good choice for this project. JupyterHub, a multi-user Hub, spawns, manages, and proxies multiple instances of the single-user Jupyter notebook server. JupyterHub can be used to serve notebooks to a class of students, a corporate data science group, or a scientific research group [5]. JupyterHub brings the power of notebooks to groups of users. It gives users access to computational environments and resources without burdening the users with installation and maintenance tasks. Users - including students, researchers, and data scientists - can get their work done in their own workspaces on shared resources which can be managed efficiently by system administrators [5].

JupyterHub allows users to interact with a computing environment through a web page. As most devices have access to a web browser, JupyterHub makes it is easy to provide and standardize the computing environment of a group of people (e.g. for a class of students or an analytics team).The Kubernetes can help us set up our own JupyterHub on a cloud and leverage the clouds scalable nature to support large groups of users.

Summary of Final Decision

The final decision for the project was to use the Django web framework with the embedded SQLite database due to its ease of use.

For the on-site coding environment, we chose to employ a JupyterHub Server managed by Azure Kubernetes Service (AKS) as it supports multiple users.
