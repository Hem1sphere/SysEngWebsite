Maintainability

The project is open-source and is available on a public repository. We aimed for the code being clear in its intent by writing necessary comments and giving clear variable names to maximise maintainability in that other developers can easily pick up and modify the source code for their purposes.

Project Management

While we assigned tasks each week and aim for an even split, there were times where the work was not well distributed. This resulted in members being unable to finish or needed more time. This caused some disruptions to the project schedule but was mitigated by mutual understanding and clear communications between members. The team was willing to help each other in times of difficulty as well.

Future Work
