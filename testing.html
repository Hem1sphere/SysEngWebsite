Unit and Integration Testing

Our tests of the application itself follows the best practices as recommended by Django and makes extensive use of the Django test suite. For each sub-application (the different Django applications e.g. users, challenges etc), tests were written for views, models and forms where applicable.

The tests for views include whether the correct template was used, whether the URL routing is correctly set up and HTTP requests such as post requests are correctly processed etc. In the tests for models, we test for correct behavior of object creation and retrieval as desired. For forms, form validation is tested.

Automated Testing

To enable automated testing, we made use of the auto-testing feature found on PyCharm. It works by running the selected test classes whenever changes to the code are made.

When making changes to a certain part of the application, we select the relevant test code to be ran under the auto-testing feature. This ensured a safe workflow - any modifications to the codebase will not break the application unnoticed.

We carried out user acceptance testing by coming up with a list of tasks a user is expected to accomplish with ease if our site is well-designed and getting their feedback upon completion of those tasks.
